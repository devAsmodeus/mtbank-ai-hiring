# Тестовые данные

Синтетические записи русской речи (edge-tts: `ru-RU-SvetlanaNeural` — оператор,
`ru-RU-DmitryNeural` — клиент). Синтетика выбрана осознанно: реальные звонки
банка — персональные данные; TTS даёт точные эталонные транскрипты (текст,
поданный в синтез) и полный контроль сценария. ТЗ явно допускает TTS.

Регенерация: `python scripts/generate_test_data.py` (нужны интернет и ffmpeg).

| Файл | Формат | Частота | Длит. | Сценарий |
|---|---|---|---|---|
| `dialog_credit_16k.wav` | WAV PCM16, моно | 16 кГц | 130 с | Диалог о кредите наличными (сценарий из ТЗ), 2 говорящих |
| `dialog_credit_8k_phone.wav` | WAV µ-law, моно | **8 кГц** | 130 с | Тот же диалог через телефонный кодек |
| `dialog_card_block.mp3` | MP3 64 кбит/с, моно | 16 кГц | 94 с | Блокировка карты; **намеренные compliance-нарушения**: запрос CVV, «гарантирую стопроцентно», давление «предложение сгорит» |
| `dialog_card_block_stereo.wav` | WAV PCM16, **стерео** | 16 кГц | 94 с | Тот же диалог, оператор — левый канал, клиент — правый (как пишут PBX) |
| `monologue_deposit.ogg` | OGG Vorbis, моно | 16 кГц | 47 с | IVR-монолог об условиях вкладов (один голос) |
| `monologue_transfer_8k.wav` | WAV µ-law, моно | **8 кГц** | 24 с | Голосовое обращение клиента о переводе |

Суммарно: **8,7 мин** (требование ТЗ — ≥5 мин, ≥5 файлов, есть 8 кГц и диалог 1+ мин).

Рядом с каждым аудио — эталонный транскрипт `<имя>.txt` в формате
`Роль: текст` (ровно тот текст, что подавался в TTS).

WER-таблица: [`wer_report.md`](wer_report.md), генерируется
`python scripts/evaluate_wer.py`.
